Playlist Chaos is a Streamlit-based smart playlist generator designed to explore mood-driven music organization and AI-assisted debugging. The app allows users to add songs with metadata (title, artist, genre, energy), dynamically build mood-based playlists, search across tracks, generate statistics, and use a “Lucky Pick” feature for random selection. The codebase was intentionally imperfect to encourage investigation and iterative improvement. Through targeted debugging, I improved playlist merge behavior to avoid unintended mutation, corrected statistical calculations for accuracy, fixed artist search to support intuitive substring matching, ensured Mixed songs are included in Lucky Pick modes, and added safe handling for empty playlists. These updates make the app’s behavior more predictable, consistent, and aligned with user expectations while preserving its exploratory design purpose.

The core concept students needed to understand was that Streamlit reruns the entire script on every interaction, so any value not stored in st.session_state will reset. Students are most likely to struggle with invisible reruns and accidentally regenerating values (like the secret number or playlist data) each time a button is clicked. AI was helpful in explaining how reruns and session state work conceptually, but it was sometimes misleading by suggesting larger refactors instead of identifying the minimal state-related fix. The most important debugging step was isolating which variables were being reinitialized on each rerun and moving them into persistent session state. To guide a student without giving the answer, I would ask them to trace what happens from top to bottom when they click a button and identify which variables are recreated. This encourages them to reason about execution flow rather than just applying a patch.
